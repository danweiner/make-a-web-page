Basics of the Web and HTML

Major Pieces of the Web

The web consists of 4 components: users, a computer with a browser, the internet, and servers.

The browser makes a request, via the internet, to servers. The internet communicates with the servers using HTTP. The servers respond with files which the browser displays.

See this video for a more in depth explanation of the web.

HTML

HTML is short for "HyperText Markup Language". HTML consists of text content, markup (tags and elements), references to other documents (images and video) and links.

A few examples of tags are bold, italics, anchor (for links) and image (this is a void tag- doesnt require a closing tag).

Tags can also have attributes. For instance, the anchor tag has the attribute href. This is used for creating links. Image tags have the attributes src (for url) and alt (for text when image doesnt load).

Inline vs. Block Elements

br (line break) is an inline element while p is a block element. Block elements form an "invisible box" around the content inside of them.

Span is inline while Div is a block element. These elements don't do anything besides contain content, but you can attach styles to them to affect the display of the content. You do this by adding CSS classes.

Creating a Structured Document with HTML

First Step

A front end web developer will receive a design markup from the graphic design team. The front end developer needs to turn that design into a webpage.

Three elements make up the conceptual design of a website:
1. HTML
2. CSS
3. Java

Tree-Like Structure

The browser builds a tree-like structure of a page. This is a branching structure that is made up of HTML elements.

HTML-CSS-DOM

HTML is like a language. It has syntax and rules. HTML tags are elements in the tree-like structure.

DOM (document object model) is the tree-like structure.

CSS creates the style of a page. It dictates how the elements will look on the page.

Boxes Everywhere

Everything on a webpage is a box. This is helpful for a front end web developer because you can rearrange these boxes if you want to change the look of a webpage. You start with the big boxes and then work towards the smaller ones.

Adding CSS Style to HTML Structure

Don't Repeat Code!

An important concept in CSS is to not repeat code. This helps programmers avoid errors and it increases consistency throughout the document.

Adding Style

In terms of structure, add doctype to the top of the document (HTML). Also, add a head tag, and put the CSS within the head tag. Then the body tag. In scratchpad, instead of a head tag, you can use a style tag for CSS.

Understanding CSS

CSS = Cascading Style Sheets. You can have several style sheets referenced in HTML. The sheets are "cascading" because the most specific rule is applied to every element.

The Box Revisited

Everything on a page is a box.

Boxes are made up of a margin, border, padding, and content. The border box element is used to make calculating box size easier.

Flex box is used for positioning boxes next to each other.
